Overview

NDAS Software OEM Localization and Branding Kit (LBK) provides OEM partners to localize and/or to change messages or text in the NDAS software.

Prerequisites

LBK requires Microsoft .NET Framework 2.0 to build the customized binary files for testing purpose. If your system does not have Microsoft .NET Framework Runtime files, you can download the package from the Microsoft Web Site:

  • http://www.microsoft.com/downloads/details.aspx?FamilyID=0856eacb-4362-4b0d-8edd-aab15c5e04f5&displaylang=en

Directories and Files

If the path of the project directory contains spaces, build.cmd will not work. Make sure to place files into the directory without spaces in its path.

  • Won't work: c:\documents and settings\user\my documents\ndas_3.30.1601_oem_lbk
  • Okay: C:\ndas_3.30.1601_oem_lbk

Root directory of the kit has directories:

base

Base Files including binary files, icons, bitmaps, images, translation base xml files.

[oem-name]

Customized data and output files

tools

Executable files to build customized binary files

[oem-name] directory has the following files:

loc.proj

MSBuild Project file. Used for creating customized binary files

loc_enu_ndasbind.xml

NDAS Bind Management Translation File (English)

loc_enu_ndasmgmt.xml

NDAS Device Management Translation File (English)

loc_enu_ndasmsg.xml

NDAS Device Management Translation File (English)

[oem-name]\resdata is the directory for customized icon files and bitmap files. You may see branded headers or icon files in this directory. All of the customizable base files are located in base\resdata. Take a look at files in base\resdata. If you see some files that you need for customization, copy them into [oem-name]\resdata and modify them.

Steps

1.   Text and Message Changes

Open a translation file (e.g. [oem-name]\loc_enu_ndasmgmt.xml) with Localization Editor (tools\locedit.exe), edit strings and save it.

2.   Binary resource modification

Replace the existing icons with customized icons in [oem-name]\resdata directory.

3.   Build the binaries

Run [oem-name]\build.cmd to build the customized binary files. It is recommended to run this command from cmd.exe to see warnings or errors which you may encounter. After the successful build, you will see files in [oem-name]\branded\i386, which are changed from the base binary files, applying translations and modified resources.

4.   See the result

You may run ndasmgmt.exe or ndasbind.exe from the output directory, if you have the same version of NDAS Software installed in your system. Otherwise, install the NDAS Software in another system and copy these output files into the system. If you see more changes are required, you may modify further and re-run the build.cmd.

5.   Submit changes

After editing and reviewing your changes, send us the final output of translations and resource data files, which are [oem-name]\*.xml and [oem-name]\resdata\*.*. And we will build your software and send the final setup package.

Notes on translating

 

  • It is critically important that things like %d and %s are not removed or changed. The comments on the right may explain what they are. When SyncBack is run %s will be replaced by a string, and %d is replaced by a number. Sometimes they are prefixed with a number, e.g. %1:d This is the order in which they are replaced, e.g. for the string "%0:s (%1:d of %2:d files)" the string is first (%0:s), a number is next (%1:d), followed by another number (%2:d).

 

  • Keep the line feeds (\n) in the same place when translating.

 

  • If a string has \" then keep those, e.g. Click to resume \"%s\"

 

  • Sometimes ampersands (&) are used for accelerator keys, e.g. menu shortcuts. For example, "&Stop" means "S" is the accelerator key. If possible, try to keep the same letter when translating.

 

  • There is no need to translate font names, e.g. if the Original String is "MS Sans Serif" or "Tahoma" then do not change it. In SyncBack the user can choose the font to use.

 

  • If something does not need translating, e.g. “%0:s V%1:d.%2:d.%3:d.%4:d %5:s” then there is no need to enter a translation string. Just leave it blank so the default English string is used.

 

  • Do not translate filenames, paths, or filename extensions, e.g. “*.exe;*.com;*.bat;*.vbs”, “*.zip”, etc.
